List of all HTML rules

When we are writing code with HTML, there are many rules that we have to follow properly. These are some examples:

  • When you create an HTML document, you need to use an .html extension for your filename.
  • Define a doctype (<!DOCTYPE html>)at the top of an HTML document.
  • An HTML document must have a head which contains the current document like a title, and a body that consists all of your main content.
  • You can't have an end tag without a start tag.
  • Tags are always surrounded by angle brackets (less-than </greater-than > characters). For example:
    <title> Learning HTML </title>
  • The second tag (the ending tag) always starts with a forward slash (/).
  • When you start an tag within another tag, you have to close that inner tag before closing the outer tag.
  • In some cases, tags have an optional attributes that use values to modify the tag's purpose.
    <input type="text">
  • Use lower case instead of uper case for the element name.
  • Commas are not acceptable inside of tags.
  • Have a nice indenting to make your code easy to read, do not start an element iside.
  • Nest your elements properly. If you want more explanation about the nesting rules, please go to this page:
    Learn how to nest elements
  • Pay attention when writing an attribute name, the page below elaborates on the attribute rules:
    See more about attribute rules
  • For high-quality code, do test your html in the data validator , and you will see what is wrong with your code.
  • Make your indenting spaces into 2, because 4 spaces may create conflicts when you push your work.
  • Do not keep everything that is totally unecessary, you can do that by questioning everything.
  • Validate your code and use markup according to its purpose.
